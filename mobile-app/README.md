# OfficePortal вЂ” mobile-app

Flutter employee app for OfficePortal HRMS.

## Status: Dart-side scaffold

This directory contains the **Dart side** of the Flutter scaffold вЂ”
`lib/`, `pubspec.yaml`, `analysis_options.yaml`, `.env.example`. The
platform-specific directories (`ios/`, `android/`, `web/`, `macos/`,
etc.) are not yet generated and must be created by running Flutter's
project generator over this skeleton.

### One-time setup

1. Install Flutter (https://docs.flutter.dev/get-started/install) and
   confirm `flutter doctor` is clean.
2. From this directory:
   ```bash
   cd mobile-app
   flutter create . --project-name officeportal_mobile --org com.officeportal
   ```
   `flutter create .` adds the missing platform dirs and a default
   `test/widget_test.dart` without overwriting your existing `lib/`,
   `pubspec.yaml`, or `analysis_options.yaml`.
3. Pull deps:
   ```bash
   flutter pub get
   ```
4. Copy the env template:
   ```bash
   cp .env.example .env
   # adjust API_URL for your local backend
   ```

### Run

```bash
# Use --dart-define-from-file so API_URL is baked in at compile time.
flutter run --dart-define-from-file=.env
```

The `API_URL` default (`http://10.0.2.2:8000`) targets the Android
emulator's loopback alias. iOS simulator: use `127.0.0.1`. Physical
device on the same Wi-Fi: use the host machine's LAN IP.

## Stack

| Layer | Library |
|---|---|
| Framework | Flutter (Material 3) |
| HTTP | Dio (single instance in `lib/core/api/api_client.dart`) |
| State | Riverpod (`flutter_riverpod`) |
| Routing | go_router with auth-aware redirects |
| Secure storage | `flutter_secure_storage` (Keychain / EncryptedSharedPreferences) |
| Env | compile-time via `--dart-define-from-file=.env` |

## Auth flow

1. App boots в†’ `AuthController._bootstrap()` reads token from
   `flutter_secure_storage`.
2. If a token exists, hit `/auth/me` to populate `AuthState.user`.
3. `GoRouter` redirects based on `AuthState.status`:
   - `loading`        в†’ `/splash`
   - `unauthenticated` в†’ `/login`
   - `authenticated`  в†’ `/dashboard`
4. Login submits roll number + password to `/auth/employee/login`,
   writes the token to secure storage, then re-runs `/auth/me`.
5. Dio interceptor attaches `Authorization: Bearer <token>` on every
   outgoing request. 401 responses clear the stored token вЂ” the
   router redirect then bounces back to `/login`.
